核心功能

两个 Tab，
一个完整的 AI 情报站

自动化的新闻聚合 + 可交互的产品对比，每天打开就能了解 AI 世界在发生什么。

📡
多源 RSS 自动抓取
同时抓取 TechCrunch AI、The Verge、VentureBeat、MIT Tech Review、OpenAI Blog、Anthropic Blog 等 6 个顶级来源，并发请求节省时间。
🛡️
假新闻过滤机制
域名白名单准入、多源交叉验证（同一新闻出现在 2+ 来源才通过）、标题相似度去重（difflib），层层过滤不可信内容。
🌏
中英双语对照
使用 googletrans 将标题和摘要实时翻译成中文，界面展示中文大标题 + 英文原文对照，翻译结果缓存 1 小时避免重复请求。
🔊
语音播报 + 声音选择
点击播放即可听 AI 朗读所有新闻，支持选择系统中文声音（优先列出中文语音），可调节语速和音调，朗读时当前新闻卡片自动高亮滚动。
📊
AI 产品能力排行
按「底层大模型 → 对话产品 → 编程工具 → Agent 平台 → 本地框架 → 图像/视频/语音」8 层分类，S/A/B 能力 Tier 评级，涵盖 40+ 款主流产品。
🏷️
NEW 动态角标
自动将今日新闻与产品关键词匹配，有相关新闻的产品实时显示红色 NEW 角标，并直接展示相关新闻链接，一眼看到行业动态。

设计思路

怎么做出来的

从一个想法到可用的桌面 App，整个过程用 Claude Code 完成，大约花了 30 分钟。

STEP 01
明确需求与技术方案
需要一个每日 AI 新闻 App：Tab 1 做产品横向对比并标注新 NEWS，Tab 2 做日更新闻并支持语音播报。系统没有 Node.js，选择 Python Flask 本地服务器 + HTML/CSS/JS 前端，在浏览器中展示，Web Speech API 负责 TTS，无需安装额外桌面框架。
Python Flask 无 Node.js 单页应用
STEP 02
新闻后端：抓取 + 过滤 + 翻译
用多线程并发抓取 6 个 RSS 源，对返回的条目做三层过滤：① 域名白名单（只信任 techcrunch.com / theverge.com / openai.com 等）② 多源交叉验证（单一来源的故事可信度低）③ difflib 标题相似度去重（同一新闻不重复展示）。翻译用 googletrans 免费库，结果缓存在内存中，1 小时 TTL。
RSS + feedparser 域名白名单 difflib 去重 googletrans 内存缓存
STEP 03
产品数据库设计
参考用户提供的「AI 完整生态图」，将产品按层次划分：底层大模型 → 对话产品 → 编程工具 → Agent 平台 → 本地框架，再追加图像/视频/语音 3 个创作类分类，共 8 类 40+ 产品。每个产品配置 keywords 数组，后端自动将今日新闻标题与 keywords 匹配，命中则返回 has_news: true 和相关新闻链接。
8 层分类 40+ 产品 关键词匹配 动态 NEW 角标
STEP 04
前端界面：深色科技风格
纯 HTML/CSS/JS 单页，无框架依赖。S/A/B Tier 徽章用渐变色区分，NEW 角标带呼吸动画，新闻卡片在朗读时自动高亮。语音控件使用 window.speechSynthesis，优先列出系统中文声音，语速/音调滑块实时生效。
纯原生 CSS Web Speech API 深色设计 无依赖前端
STEP 05
一键启动脚本
launch.py 自动做三件事：pip install 所有依赖 → 启动 Flask 服务器监听 7788 端口 → 1.5 秒后用 webbrowser.open 自动打开浏览器。用户只需要在终端运行一行命令。
自动安装依赖 自动开浏览器 一行启动

快速启动

三步跑起来

只需要 macOS 自带的 Python 3，无需安装 Node.js 或其他运行时。

$ git clone https://github.com/yusizou-commits/ai-news-app.git # 下载代码
$ cd ai-news-app # 进入目录
$ python3 launch.py # 自动安装依赖、启动服务器、打开浏览器

⚠️ 终端窗口需保持开启。第一次加载新闻约需 15-30 秒（需联网翻译）。
