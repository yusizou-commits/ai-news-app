FEATURE 03
🛡️ 三层假新闻过滤
新闻抓取后经过三道过滤，确保展示的每一条都来自可信来源、内容不重复、多源验证。过滤逻辑在后端 Python 运行，前端只拿到经过筛选的结果。
▸第 1 层：域名白名单，只信任 techcrunch.com / theverge.com / openai.com / anthropic.com 等 18 个顶级媒体
▸第 2 层：标题相似度去重（difflib），同一故事不重复展示
▸第 3 层：多源计数，出现在 2+ 来源的故事额外标注"多源验证"
🛡️ 新闻过滤流水线
①
域名白名单过滤
只保留来自 18 个可信域名的条目
techcrunch.com ✓ openai.com ✓ theverge.com ✓ medium.com ✗ twitter.com ✗ reddit.com ✗
②
标题相似度去重（difflib）
相似度 > 60% 的条目合并，只保留最早一条
"OpenAI 发布新模型" ✓ 保留 "OpenAI 推出新 AI 模型" ✗ 去重
③
多源验证标注
同一故事出现在 2+ 来源时标注"多源验证"
✓ 多源验证（TechCrunch + The Verge） ✓ 可信来源（单源）
✓
最终结果
6 个来源 ~90 条原始条目 → 过滤后约 17 条高质量新闻
抓取：~90 条 过滤掉：~73 条 保留：17 条 ✓
FEATURE 04
🌏 中英双语 + 1 小时缓存
抓取到的英文新闻通过 googletrans 翻译成中文，标题和摘要都会被翻译。翻译结果存入内存缓存，1 小时内不重复请求，刷新按钮可手动清除缓存立即重新抓取。
▸标题中文在上（字体大），英文原标题在下（小字斜体）
▸摘要默认显示中文，鼠标悬停展开英文原文
▸6 个来源并发抓取，总用时约 15-30 秒
▸缓存 TTL 1 小时，再次打开秒速加载

核心特性

一句话总结每个功能

📡
多源并发抓取
6 个顶级 AI 媒体 RSS 同时抓取，Python 多线程处理，第一次加载约 15-30 秒，之后缓存秒速响应。
🛡️
三层假新闻过滤
域名白名单 → difflib 标题去重 → 多源计数验证，~90 条原始条目过滤为 ~17 条高质量新闻。
🌏
中英双语对照
googletrans 免费翻译，标题和摘要全部汉化，界面同时保留英文原文供对照，悬停展开。
🔊
语音播报
浏览器原生 Web Speech API，优先调取系统中文语音，朗读时当前卡片高亮，支持语速/音调调节。
📊
8 类 40+ 产品排行
参照「AI 完整生态图」按层次分类，S/A/B 三级能力评分，覆盖大模型/编程/Agent/图像/视频/语音。
🏷️
NEW 动态角标
产品关键词与当日新闻实时匹配，命中则显示红色呼吸动画 NEW 角标，直接展示相关新闻链接。

STEP 01
需求 → 技术方案
需要播放 AI 新闻的桌面 App，系统无 Node.js，选 Python Flask 本地服务器 + HTML/CSS/JS 前端，Web Speech API 负责 TTS，浏览器里当"桌面窗口"用。
Python Flask无需 Node.jsWeb Speech API
STEP 02
新闻后端：抓取 + 过滤 + 翻译
多线程并发抓 6 个 RSS 源，三层过滤（域名白名单 + difflib 去重 + 多源验证），googletrans 翻译结果内存缓存 1 小时。
feedparserdifflibgoogletrans内存缓存
STEP 03
产品数据库 — 参照用户提供的生态图
用户提供了「AI 完整生态图」HTML 文件，包含底层大模型/对话产品/编程工具/Agent 平台/本地框架五层结构。以此为基础扩展为 8 类，追加图像/视频/语音，每个产品配 keywords 数组实现新闻联动。
8 层分类40+ 产品关键词匹配

纯原生 HTML/CSS/JS，无框架。S/A/B 渐变徽章，NEW 呼吸动画，朗读高亮，声音列表优先中文语音。修复了 speechSynthesis 未初始化导致 JS 报错的 bug。

三步跑起来

只需要 macOS 自带的 Python 3，无需安装 Node.js。

$git clone https://github.com/yusizou-commits/ai-news-app.git# 下载代码
$cd ai-news-app# 进入目录
$python3 launch.py# 自动安装依赖、启动服务器、打开浏览器

⚠️ 终端窗口需保持开启。第一次加载新闻约需 15-30 秒（需联网翻译）。
